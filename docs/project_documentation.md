# Документация Проекта: Нейросеть для распознавания эмоций по голосу

Настоящий документ описывает устройство и архитектуру разработанной подсистемы Machine Learning (ML), которая принимает на вход аудиофайлы и предсказывает эмоциональное состояние человека. Документ структурирован по ключевым этапам: от постановки задачи до интеграции в Web-приложение.

---

## 1. Постановка задачи (Problem Statement)
Основная цель проекта — разработка интеллектуального алгоритма (нейросети) на базе архитектуры Deep Learning, способного классифицировать сырой аудиосигнал на 7 эмоциональных классов: `Angry`, `Disgusted`, `Fearful`, `Happy`, `Neutral`, `Sad`, `Suprised`.
Данная задача относится к классу **Audio Pattern Recognition** (Распознавание акустических паттернов) и является сложной из-за высокой размерности необработанного звука. Чтобы снизить размерность и выделить полезные признаки, мы переходим из аудио-домена в визуальный домен (обрабатываем звук как картинку).

---

## 2. Методология (Methodology)

### 2.1 Извлечение признаков (Feature Extraction)
Вместо того чтобы подавать в нейросеть сырую волну (Raw Waveform), алгоритм использует библиотеку `librosa` для преобразования звука в **Мел-спектрограмму (Mel-Spectrogram)**. 
- **Что это такое:** Это двумерное изображение (матрица), где ось X — это время, ось Y — это частоты (переведенные в нелинейную мел-шкалу, имитирующую слух человека), а цвет/яркость — это амплитуда (громкость) частоты.
- **Предобработка:** Все аудиофайлы строжайше нормализуются: 
  1. Приводятся к единой длине (padding или обрезка до 3 секунд).
  2. Нормализация मीन/ско (Z-score normalization), чтобы модель не зависела от громкости разных микрофонов.

### 2.2 Кросс-валидация (K-Fold Cross-Validation)
Чтобы гарантировать, что нейросеть действительно выучила паттерны эмоций, а не просто зазубрила конкретные голоса из датасета, применяется строгая методика **5-Fold Stratified Cross-Validation**.
Весь датасет случайным образом делится на 5 равных кусков. Модель поочередно обучается на 4 кусках и сдаёт "экзамен" на 5-ом (ранее невиданном) куске. Этот процесс повторяется 5 раз. Итоговая точность проекта — это средняя точность по всем 5 экзаменам. Модель, показавшая наивысший результат на валидации, сохраняется (`model.pt`) и идет в продакшен.

---

## 3. Архитектура модели (Model Architecture)
Для решения задачи спроектирована гибридная архитектура **CNN + LSTM**, реализованная на фреймворке `PyTorch`. Эта архитектура является "золотым стандартом" для анализа последовательных данных вроде звука.

1. **Сверточная часть (Convolutional Neural Network - CNN)**:
   - Состоит из трех блоков `Conv2D -> BatchNorm2d -> ReLU -> MaxPool2d`.
   - Задача этой части: работать со спектрограммой как с картинкой. Сверточные фильтры (размером 3x3) проходят по спектрограмме и извлекают локальные признаки: резкие скачки интонации, наличие шумов или "дрожь" в голосе. `BatchNorm` ускоряет сходимость, а `MaxPool` сжимает лишнюю размерность.
2. **Рекуррентная часть (Long Short-Term Memory - LSTM)**:
   - Сжатые признаки из CNN вытягиваются в последовательность и передаются в слой `LSTM`.
   - Задача этой части: анализ того, как фичи меняются *во времени*. Эмоция (например, страх) — это не просто один звук, это динамика (например, от тихого дрожания до крика). LSTM обладает "памятью" и идеально улавливает этот временной контекст.
3. **Классификатор (Linear Classifier)**:
   - Скрытое состояние (Hidden state) последнего шага LSTM прогоняется через слой `Dropout` (для защиты от переобучения) и финальный линейный слой (`nn.Linear`), который делает предсказание для 7 классов эмоций.

---

## 4. Интеграция в Web-приложение (Practical Application Integration)
Важнейшая часть проекта — доведение ML-модели до уровня Production-Ready сервиса, которым может пользоваться клиентская часть (Frontend). 

1. **Микросервис Model Service**: Нейросеть упакована в независимый веб-сервис на базе асинхронного фреймворка `FastAPI`. 
2. **Singleton паттерн**: При запуске сервера веса модели (файл `model.pt` объемом несколько мегабайт) загружаются в оперативную (или видео) память **один раз**. Это позволяет избежать задержек при обработке запросов пользователей.
3. **API Contract (Обмен данными)**: Пользовательский Frontend отправляет HTTP POST-запрос с аудиофайлом на маршрут `/inference`. Сервис-предсказатель (`predictor.py`) перехватывает этот файл прямо в оперативной памяти (без сохранения на диск с помощью временных буферов), мгновенно применяет нормализацию, прогоняет через обученную сеть CNN+LSTM и возвращает ответ в строгом JSON-формате:

```json
{
  "predicted_emotion": "happy",
  "confidence": 0.912,
  "probabilities": {
    "angry": 0.015,
    "disgusted": 0.005,
    "fearful": 0.010,
    "happy": 0.912,
    "neutral": 0...
```

Такая архитектура позволяет горизонтально масштабировать (увеличивать количество копий) сервис нейронной сети отдельно от остального сайта, что полностью соответствует современным индустриальным практикам масштабирования (Deep Learning Microservices).
