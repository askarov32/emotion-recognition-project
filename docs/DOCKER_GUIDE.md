# 🐳 Инструкция по сборке и развертыванию через Docker

Данный гайд предназначен для разработчиков, преподавателей и всех тех участников проекта, которым требуется настроить и использовать API-эндпоинт для работы с моделью. 

Docker позволяет изолировать нейросеть (зависимости, PyTorch, C++ библиотеки звука) так, чтобы она работала на любом компьютере *ровно таким же образом без установки Python*.

---

## 1. Откуда берется Endpoint и как он "открывается" в мир?
Сервис предсказаний (`model_service`) физически крутится внутри замкнутого контейнера (виртуальной машины). 
По умолчанию никто "снаружи" (например, ваш браузер или Frontend) не имеет к нему доступа.

Но в конфигурационном файле `docker-compose.yml` (в корне вашего проекта) прописано "пробрасывание портов" (Port Mapping). 
Код внутри контейнера ожидает запросы на порту `8001`. `docker-compose` связывает порт `8001` вашего физического компьютера (Localhost) с портом `8001` этого контейнера. 

Поэтому после запуска Docker, **глобальный API-контракт** станет доступен по адресу:
👉 `http://127.0.0.1:8001/inference` (или `http://localhost:8001/inference`)

Сюда любая клиентская часть (Frontend) может слать `POST`-запросы с аудиофайлами.

---

## 2. Предварительные требования для проверяющего
1. Скачать и установить библиотеку **Docker Desktop** c официального сайта (для Windows/Mac).
2. Запустить **Docker Desktop** (он должен работать в фоне, в системном трее должен гореть зеленый значок).
3. Убедиться, что обученная модель (`model.pt`) и файл конфигурации (`meta.json`) лежат в папке `model_service/weights/`. *(Docker скопирует этот "мозг" внутрь себя при сборке).*

---

## 3. Запуск проекта (В один клик)
Проверяющему или члену команды достаточно открыть любой терминал (CMD/PowerShell) в корне всей папки `emotion-recognition-project` и ввести **всего одну команду**:

```bash
docker-compose up --build
```

### Что произойдет дальше:
1. `docker-compose` прочитает ваш файл проекта.
2. Он скачает чистую операционную систему Linux.
3. Он самостоятельно установит в неё нужный Python, установит "тяжелый" `PyTorch` и другие зависимости из `requirements.txt`.
4. Он скопирует внутрь код проекта и файл с весами (`model.pt`).
5. Он запустит команду веб-сервера `uvicorn`, которая откроет эндпоинт.

Вы увидите в консоли строки запуска. Как только появится сообщение об успешном старте (например: *Uvicorn running on http://0.0.0.0:8001*), сервер готов к принятию аудио.

---

## 4. Как протестировать Endpoint "снаружи"
Сервер отвечает только на POST-запросы `multipart/form-data`. Любой желающий, у кого запущен ваш Docker, может убедиться, что всё работает, отправив аудиофайл.

Например, в новом терминале с помощью утилиты `cURL`:
```bash
curl -X POST "http://127.0.0.1:8001/inference" -F "audio=@C:/путь/к/вашему/тестовому/файлу.wav"
```
Ответом будет заветный JSON:
```json
{
  "predicted_emotion": "happy",
  "confidence": 0.94,
  "probabilities": { ... }
}
```

---

## 5. Как выключить проект
Чтобы правильно погасить сервер нейросети, находясь в терминале с бегущими логами Docker, нажмите **`Ctrl + C`**, и контейнер корректно закроет порты и завершит работу. 
В следующий раз команду `--build` писать не нужно: достаточно просто написать `docker-compose up`.
