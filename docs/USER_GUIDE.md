# 🚀 Полное руководство по запуску: Нейросеть распознавания эмоций

Данный гайд проведёт вас от чистого компьютера с исходным кодом до полностью обученного и работающего веб-сервера (API) нейросети.

---

## 🛑 Шаг 1: Подготовка окружения
Прежде чем что-либо запускать, убедитесь, что библиотека PyTorch видит вашу видеокарту (CUDA), а в папке с вашим датасетом лежат папки эмоций.
- Убедитесь, что внутри `C:\Users\Telmurius\python_projects\emotion-recognition-project\data\Emotions` лежат ваши 7 папок (`Angry`, `Happy` и т.д.) с аудиофайлами `.wav`.
- Откройте терминал `PowerShell` в PyCharm и убедитесь, что вы находитесь в корне проекта (`C:\Users\Telmurius\python_projects\emotion-recognition-project`).

---

## 🧠 Шаг 2: Обучение модели (Training)
Это самый важный этап. Без него ваш сервер будет выдавать "магические" 14.2% для всех эмоций (так как модель будет пустой).

1. В открытом терминале (в корне проекта) запустите скрипт:
   ```powershell
   python model_service/training/train.py
   ```
2. **Как это работает:**
   - Скрипт начнет сканировать `.wav` файлы и создавать рядом с ними кэш `.pt` (для колоссального ускорения). Это займет 1-2 минуты только один раз.
   - Далее начнется обучение `Фолд 1 / 5` на видеокарте.
   - Вы будете видеть растущие проценты `Val Acc` (точность).
   - Как только скрипт найдет лучшую точность (например, 73%), он **автоматически сохранит её** в `model_service/weights/model.pt`.
3. Можете дождаться конца всех 40 эпох, а можете прервать обучение кнопками **`Ctrl + C`**, если вас уже устраивает сохраненная точность. *(Главное — убедитесь, что файл `model.pt` появился в папке `weights`)*.

---

## 🌐 Шаг 3: Запуск Сервера (API Backend)
После успешного обучения (и наличия файла `model.pt`), вы можете поднять сам веб-сервер, который будет "слушать" запросы от вашего фронтенда или сайта.

1. Остановите обучение (Ctrl+C) или откройте новую вкладку терминала.
2. Обязательно зайдите в папку микросервиса:
   ```powershell
   cd model_service
   ```
3. Запустите веб-сервер (uvicorn):
   ```powershell
   uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
   ```
4. Если всё успешно, сервер напишет зеленым цветом: 
   `[predictor] Обученные веса загружены из ...model.pt`
   *(Если вы видите `ВНИМАНИЕ: Файл model.pt отсутствует`, вернитесь к Шагу 2)*.

---

## 🧪 Шаг 4: Проверка работы нейросети (Тестирование)
Сервер "виснет" в режиме ожидания запросов. Чтобы убедиться, что он может "скушать" чужой `.wav` файл с вашего будущего сайта и ответить JSON-ом, сделаем тестовый запрос.

1. Оставьте терминал с сервером из `Шага 3` открытым (пусть висит).
2. Откройте **вкладку Нового терминала (Local)** в PyCharm (кнопка `+` на панели терминала). Убедитесь, что новый терминал открылся снова в корне проекта.
3. Запустите симулятор отправки аудио с сайта:
   ```powershell
   python test_request.py
   ```
4. Если всё сделано верно, вы мгновенно получите "сырой" ответ нейросети в формате JSON (API Contract), где на одну из эмоций будет огромная вероятность (например, `0.941...`), а на остальные — крошечная. Всё работает безупречно!

---

### 💡 Полезные советы (Troubleshooting):
- **Как сбросить обучение "в ноль":**
  Выполните в PowerShell (находясь в корне):
  ```powershell
  Get-ChildItem -Path "data\Emotions" -Filter *.pt -Recurse | Remove-Item -Force
  ```
  и удалите папку `model_service/weights`. Затем начните с Шага 2.
- **Всё подсвечено красным в PyCharm:**
  Ваша среда не видит глобальные библиотеки. Зайдите в `Settings (Ctrl + Alt + S) -> Python Interpreter -> Add Local Interpreter -> System Interpreter` и выберите ваш системный `Python 3.13`. Затем нажмите Apply, и PyCharm всё подхватит.
